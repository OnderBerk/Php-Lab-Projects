Apply and observe the results of following regular expressions written in red

  1. the

    Find all occurrences of three characters having "t", "h" and "e"

  2. the|The

    Pipe, |, operator means "OR"

  3. the|The|Some

    Find all occurrences of "the", "The" or "Some"

  4. (t|T)he

    ( ) : capture group, it contains lowercase OR uppercase t and then "h" and "e"

  5. [tT]he

    [ ] : options, Square bracket represents a single character which will contain the characters within square bracket. In this example, the character set in square bracket is t and T.

  6. [0123456789]

    Find all single digits.

  7. [0-9]

    Find all single digits.

  8. \d

    Find all single digits, [0-9] is exactly the same with \d

  9. [^0-9]

    Find all non single digits, ^ in square bracket means "not"

  10. \D

    Find all non single digits, [^0-9] is equal to \D (capital D)

  11. [a-z]

    Find all lowercase letters

  12. [A-Z]

    Find all uppercase letters

  13. [a-zA-Z]

    Find all letters (lowercase or uppercase)

  14. [a-zA-Z0-9]

    Find all alphanumeric characters.

  15. [a-zA-Z0-9_]

    Find all alphanumeric characters and underscore, this character set is called "word" characters.

  16. \w

    Word characters in short, [a-zA-Z0-9_] is equal to \w

  17. \W

    Non Word characters in short, [^a-zA-Z0-9_] is equal to \W

  18. [ \t\n]

    Find white spaces. White space contains space, tab and new line.

  19. \s

    Find white spaces. [ \t\n] is equal to \s

  20. \S

    Find non white space characters. [^ \t\n] is equal to \S

  21. .

    Find any single characters (except new line \n)

  22. \.

    Find dot character itself. For meta characters, use backslash for escape.

  23. \/\/

    Find consecutive two forward slashes //

  24. \d\d\d\d

    Find group of four digits.

  25. \d{4}

    { } : quantifier. Find group of four digits. (compact form)

  26. \d{4,}

    { } : quantifier. Find group of four or more digits.

  27. \d{2,4}

    { } : quantifier. Find group of two, three and four digits.

  28. \d{1,}

    { } : quantifier. Find one or more digits.

  29. \d+

    { } : quantifier. Find one or more digits. {1,} is the same with "+"

  30. \d{0,}

    { } : quantifier. Find zero or more digits.

  31. \d*

    { } : quantifier. Find one or more digits. {0,} is the same with "*"

  32. [A-Z]{2}\d{4}-\d{2}

    Find serial numbers (two capital, then four digits, then hyphen, the two digits).

  33. \b[A-Z]{2}\d{4}-\d{2}\b

    \b : word boundary. Find serial numbers as a word, not a part of a bigger string.

  34. ^[A-Z]{2}\d{4}-\d{2}\b

    ^ : if caret is not within square bracket, it shows a position. It is the first word boundary position. It finds all lines starting with a serial number.

  35. \b[A-Z]{2}\d{4}-\d{2}$

    $ : it shows a position. It is the last word boundary position in a line. It finds all lines ending with a serial number.

  36. ^[A-Z]{2}\d{4}-\d{2}$

    ^pattern$ shows exact matching. Find all lines having only one serial number.

  37. \b(?:int|float)\s+(\w+)\s*=\s*(\d+)\b

    * : a quantifier for zero or more occurrences. \s* means zero or more occurrences of white space character
    (?: ) is used for grouping, but regex does not capture(save) the group's content. It is more efficient than capture group ( ) in terms of memory usage.
    It finds all float or int variable definitions, it captures all variable names and their values for all full matches.

  38. \b\d{1,2}:\d{1,2}(:\d{1,2})?\b

    ? : means optional, a quantifier for zero or one occurrence. It finds hours and minutes separed by colon (:), the second at the end is optional. 12:34:56 or 12:34, the last part is optional.

  39. <b>(.*)<\/b>

    Find the content of b tags.

  40. href="(.*?)"

    ? after *, or + means lazy mode. It finds the smallest match to the regex. By default, * and + execute in greedy mode, means it finds the largest match. Test with or without ? after *.

  41. <(\w+)\s*.*?>.*<\/\1>

    \1 is a back reference. It references to the first capture group. The first capture group is (\w+) that shows the opening tag. In this way, the closing tag must be the same with opening tag.

  42. ^\s*\/\/(.*)|^\s*#(.*)

    Find all comment lines (starting with // or #)

  43. ^\s*(?:\/\/|#)(.*)

    Find all comment lines (starting with // or #) - Compact form
